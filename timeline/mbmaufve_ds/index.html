At this point I should probably mention Respec, respecing a character midgame, where you’re allowed to throw away some or all of what you’ve decided on in the past and resend your skill points, reselect perks, do whatever. This is relatively new in RPG design.

You didn’t see this in, I want to say, any game 20 or 30 years ago. If you did, it was very few and it’s become a lot more common now. Personally, as a designer, I feel this is kind of a Band-Aid because once it’s put in a game, you don’t have to worry about being able to finish with a particular kind of character. But if it’s added because you feel you needed to add it and then the design subsequently, you’re like, eh, I’m going to go ahead and make characters that can’t finish. I think that’s bad, because you could have made it so characters could finish, but you — and that’s not why you added Respec, though you added it because you expected it, and now you need it. So, in summary, let me just say to the original person who asked — as I developed as a game designer, I recognized that there was a PCT between me and the players — that it would be a bug — a design bug, not a code bug — if a character build couldn’t finish the main story of the game. And this bug is as bad as any crash bug, as bad as any, you know, reduced frame rate bug. It’s just happens to be a design bug and not a code bug, and I think it needs to be fixed, because one of the mantras in my games is: do whatever you want.
